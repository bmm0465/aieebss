## 특수 케이스 처리

코드에서 다음 특수 케이스들이 처리됩니다:

1. **가족 관계 표현**
   - "dad", "father" → `dad.png` 또는 `grandfather.png`
   - "mom", "mother" → `mom.png` 또는 `grandmother.png`
   - "brother" → `brother.png`
   - "sister" → `sister.png`

2. **동작 표현**
   - "sit down" → `sit.png`
   - "stand up" → `stand.png`
   - "open the door" → `door.png` 또는 `open.png`
   - "close the door" → `door.png` 또는 `close.png`

3. **이미지가 없는 표현**
   - "Thank you", "You're welcome", "That's okay", "Here you are" 등은 감정/상황 표현이므로 이미지가 필요 없습니다.
   - 이러한 표현들은 문항 생성 시 필터링됩니다.

### 📝 참고사항

1. **이미지 매칭 로직**: 코드는 문장/어구에서 핵심 명사나 동작 단어를 자동으로 추출하여 이미지와 매칭합니다.

2. **보기 이미지**: 각 문항의 보기는 3개이며, 정답 이미지와 2개의 오답 이미지로 구성됩니다. 오답 이미지는 같은 카테고리의 다른 단어들 중에서 선택됩니다.

3. **문항 생성 순서**: 
   - 단어 문항 (10개)
   - 어구 문항 (10개)
   - 문장 문항 (10개)
   - 총 30개 문항이 **단어 → 어구 → 문장** 순서로 순환하며 출제됩니다.

4. **이미지 파일명**: 모든 이미지는 소문자 단어명으로 저장되어야 합니다 (예: `apple.png`, `big.png`).
