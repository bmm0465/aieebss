📊 피드백 수집 방법

1. 설문 조사

  • 학생: 구두 또는 간단한 체크리스트 (교사가 대신 작성)
  • 교사: 온라인 설문 또는 서면 설문

2. 관찰 기록

  • 관찰 일지: 교사가 평가 진행 중 관찰한 내용 기록
  • 비디오 녹화 (동의 하에): 학생의 평가 진행 과정 녹화 후 분석

3. 인터뷰

학생 인터뷰 (10분 내외):

  • 가장 어려웠던 부분은?
  • 가장 쉬웠던 부분은?
  • 개선했으면 하는 점은?

교사 인터뷰 (20-30분):

평가 결과 분석 및 종합 의견

4. 데이터 분석

  • 시스템 로그: 오류 발생 빈도, 응답 시간 등
  • 결과 분석: 정답률, 완료율, 시간 분포 등

📅 테스트 일정 예시

1주차: 사전 준비

  • 교사 대상 평가 방법 안내
  • 테스트 환경 점검
  • 학생 계정 생성

2주차: 파일럿 테스트

  • Day 1: 학생 3-5명 평가 (약 30-40분/명)
  • Day 2: 학생 3-5명 평가
  • Day 3: 평가 결과 검토 및 초기 피드백 수집

3주차: 피드백 분석

  • 설문 조사 수집
  • 인터뷰 진행
  • 데이터 분석
  • 개선 사항 도출

🎯 핵심 확인 사항

기술적 측면

  • ✅ 시스템 안정성 (오류, 멈춤, 로딩)
  • ✅ 음성 인식 정확도
  • ✅ 오디오/비디오 재생 품질
  • ✅ 데이터 저장 및 조회

교육적 측면

  • ✅ 평가 내용의 타당도
  • ✅ 난이도의 적절성
  • ✅ 결과의 활용 가능성
  • ✅ 학습 진단 효과

사용자 경험 측면

  • ✅ 사용 편의성
  • ✅ 접근성
  • ✅ 시각적 디자인
  • ✅ 오류 처리 및 안내
